# 通用预训练实验报告

## 实验目标

先在 8 个 UEA 多变量时序数据集上进行通用预训练，再迁移到 NATOPS 单数据集微调，验证跨数据集预训练是否能提升 NATOPS 检索性能。

---

## 阶段一：通用预训练（Universal Pre-training）

### 配置（`universal_mix_v1.json`）

| 参数 | 值 |
|------|----|
| 数据集 | NATOPS + BasicMotions + RacketSports + Epilepsy + ArticularyWordRecognition + UWaveGestureLibrary + JapaneseVowels + Libras |
| 样本总数 | 1353 条（训练集） |
| 模型 | Qwen3.5-4B + CNN TSEncoder + Projector，LoRA r=8 |
| Epochs | 40 |
| LR | 5e-5 |
| Batch size | 18（MultiDatasetBatchSampler，同批次同数据集） |
| alignment_text_mode | context（无 CoT） |
| softclt | 开启，sigma=1.0 |
| GPU | 4090-01 GPU 0 |

### 训练结果

| 指标 | 值 |
|------|----|
| 初始 loss | 2.84 |
| 最终 loss（ep40） | 1.54 |
| 总训练时长 | ~4h |
| 输出 checkpoint | `checkpoints_universal_v1/model_last.pt`（106 MB） |

无 Triton OOB crash（服务器环境下 `fla.ops.gated_delta_rule.naive` 不可用，patch 未实际触发；`max_length=320` 也使 token 长度低于 Triton 阈值）。

## 阶段二：NATOPS 微调（Fine-tuning from Universal Checkpoint）

### 配置（`natops_universal_finetune_v1.json`）

| 参数 | 值 |
|------|----|
| 起点 | `checkpoints_universal_v1/model_last.pt` |
| 数据集 | NATOPS（180 训练样本，6 类） |
| Epochs | 40 |
| LR | **1e-4**（过高，根因） |
| warmup_epochs | 3 |
| memory_bank_size | 90 |
| GPU | 4090-01 GPU 1 |

### 训练过程异常

- **Loss spike（ep3 → ep4）**：warmup 结束时 loss 从 1.77 跳升至 3.86
- **Train LOO 过拟合**：best train LOO = 94.44%（ep ~20）
- **LOO-Test gap = 14.44%**：训练集 94.44% vs 测试集 80.00%

| 版本 | Best Train LOO | Test Acc | 原因 |
|------|---------------|---------|------|
| v9（直接训练） | 84.44% | **83.89%** | 从随机初始化，LR=1e-4 合适 |
| universal→finetune v1 | 94.44% | 80.00% | 预训练权重 + LR=1e-4 → loss spike + 过拟合 |

---

## Recall@K 对比

### v9（直接 NATOPS 训练，当前最优）

| K | P@K | Hit@K | Perfect@K |
|---|-----|-------|-----------|
| 1 | **84.44%** | 84.44% | 84.44% |
| 2 | 80.37% | 95.00% | 65.56% |
| 3 | 80.37% | 97.22% | 63.33% |
| 4 | 80.83% | 98.33% | 63.89% |
| 5 | 81.39% | **98.33%** | 64.44% |

### universal→finetune v1（过拟合版本）

| K | P@K | Hit@K | Perfect@K |
|---|-----|-------|-----------|
| 1 | 81.67% | 81.67% | 81.67% |
| 2 | 81.39% | 84.44% | 78.33% |
| 3 | 81.48% | 85.56% | 77.22% |
| 4 | 81.94% | 87.22% | 77.22% |
| 5 | 82.11% | 88.33% | **75.00%** |

**关键观察：**
- P@1 下降 2.77%（81.67% vs 84.44%）
- Hit@5 大幅下降（88.33% vs 98.33%）：预训练模型对难例泛化差
- Perfect@5 更高（75% vs 64.4%）：对简单样本更确定，但对难例完全错误
- 过拟合特征明显：训练集性能虚高（94.44%），测试集实际退步

---

## 根因分析

| 问题 | 根因 |
|------|------|
| Loss spike at ep4 | LR=1e-4 对预训练权重过于激进，warmup 结束时 LR 满值瞬间破坏已对齐的语义空间 |
| Train-test gap 14.44% | LLM LoRA 层快速过拟合 180 个 NATOPS 样本（每类 30 个），需独立低 LR |
| Hit@5 崩溃 | TS Encoder 过度适配训练集分布，丢失通用预训练学到的跨类边界 |

---

## 改进方案（v2）

### 核心改动：Layer-wise LR Decay + 降低基础 LR

| 参数 | v1 | v2 |
|------|----|----|
| base LR | 1e-4 | **3e-5** |
| warmup_epochs | 3 | **5** |
| llm_lr_scale | 1.0（无区分） | **0.1**（LLM LoRA 使用 3e-6） |
| TS encoder LR | 1e-4 | 3e-5（全量） |
| save_dir | checkpoints_natops_universal_v1 | checkpoints_natops_universal_v2 |

**动机：**
- `ts_encoder + projector`：在 1353 样本上预训练但 NATOPS 专属特征未充分学习，需要充分的 LR 来适应
- `LLM LoRA`：已学好跨数据集语义对齐，用低 LR (×0.1) 保持稳定，避免过拟合 180 个样本

---

## 后续计划

1. **v2 微调**：3e-5 + warmup=5 + llm_lr_scale=0.1，目标 test acc > 84%
2. **通用预训练质量评估**：对 BasicMotions / RacketSports 等其他数据集测 P@1，确认通用预训练的实际泛化性
3. **数据增强强化**：通用训练阶段对小数据集（Libras 180, BasicMotions 80）加强 channel dropout + jitter
