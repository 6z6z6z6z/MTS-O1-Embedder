# MTS-O1-Embedder 项目进展与思路整理

**整理日期**：2026-03-23

---

## 一、项目目标

**核心问题**：能否让大语言模型（LLM）理解多变量时间序列（MTS），并以此构建高质量的检索嵌入？

具体来说，将 CNN 编码的时间序列 token 注入 Qwen3.5-4B 的隐空间，通过对比学习训练出一个嵌入模型，使得相同类别的时间序列在嵌入空间中聚拢，不同类别的时间序列相互分离。最终用于**时间序列检索**（给定查询序列，从 gallery 中找到最相似的 k 个）和 **kNN 分类**。

**验证数据集**：NATOPS（海军航空训练操作规程手势数据集）
- 24 通道 IMU 传感器，时序长度 T=51，6 类手势，180 训练样本 + 180 测试样本
- 每类仅 30 个训练样本，是一个小样本学习场景

---

## 二、系统架构（当前 v18）

```
输入: [C=24, T=51] 时间序列
        ↓
TSEncoder (CNN)
  Stem Conv (stride=2) → T≈25
  ResBlock1D × 2
  Conv Patching (patch_size=5) → P=6 个 patch
  输出: [B, C×P=144, 128]
        ↓
Projector (128 → 3584)
  Linear(128→3584) + GELU + LayerNorm
  输出: [B, 144, 3584]  ← 作为 TS tokens 插入 LLM
        ↓
Qwen3.5-4B (LoRA, r=8)
  Prompt: "Context: {手势描述}\nTask: Analyze patterns.\nData: [TS tokens 144个]"
  TS tokens 在序列末尾（因果注意力中，尾部 token 可见所有文本）
        ↓
Pooling: ts_tokens mean pooling
  对 TS token 位置的 hidden states 取均值
        ↓
final_norm (LayerNorm) → final_projection (Linear 3584→1024)
        ↓
输出: [B, 1024] 归一化嵌入向量
```

**检索**：HybridRetriever，alpha=0.9（90% 余弦嵌入相似度 + 10% DTW 结构相似度）

## 三、实验历程（v9 → v18）

### 3.1 里程碑版本

| 版本 | Train LOO | Test Acc | Test F1 | 核心改动 | 结论 |
|------|-----------|---------|---------|---------|------|
| **v9** | 84.44% | 83.89% | 81.23% | CNN+context对称+SoftCLT+EMA+ProtoNCE | **原始最优基准** |
| v10_cot | 80.56% | 80.56% | 79.57% | 加入 CoT（full 模式） | CoT 引入噪声，退步 |
| v10_patch | 68.33% | 71.11% | 69.83% | PatchTokenizer | instance norm 破坏振幅信息，退步 |
| v11 | 56.67%（崩溃） | — | — | P1-4+P7，但用 last pooling | TS 在末尾 + last pooling = 池化稀疏 vision_end，崩溃 |
| **v12** | 81.67% | **83.89%** | 80.72% | 全部系统性修复（P1-4+P7） | 与 v9 持平，泛化差距改善 |
| v13 | 85.56% | 81.67% | 79.61% | +Projector 瘦身+ChannelMixer | 参数减少但效果退步 |
| v14 | 76.67% | 82.78% | 79.43% | +窗口切片时间增强 | 增强幅度过大，退步 |
| v15 | 75.00% | 81.11% | 78.89% | teacher_thought 作 context | 实例级文本与对比损失矛盾，退步 |
| v16 | 77.78% | 82.22% | 81.08% | 类级固定描述（CLIP 类比） | 文本 shortcut，未学时序，退步 |
| v17 | 68.89%（不稳） | 82.22% | 79.96% | 双视图增强+对角注入 bug | 自对比权重 50%，退化为 SimCLR，发散 |
| **v18** | 81.67% | **85.56%** | **85.65%** | 双视图增强（label mode 修复） | **当前最优** |

### 3.2 关键转折点

**v9→v12（系统性修复）**：四项基础改进（内存库冷启动冲洗、TS 移至末尾、LayerNorm、ts_tokens pooling）合并后，测试精度维持 83.89%，但训练-测试 gap 从 +0.55% 改善到 -2.22%，过拟合明显减少。这说明这些改进提升了**泛化能力**而非记忆训练数据。

**v12→v18（双视图增强）**：核心想法是 EMA gallery 存放"干净"的弱增强嵌入，让 query 对齐的目标更稳定。最终测试准确率从 83.89% 提升到 85.56%（+1.67%），是迄今最大的单次提升。

## 五、与纯时间序列基准的对比

在同等评估条件下（gallery=train, query=test, k=5）对 5 种纯 TS 方法的完整测试：

| 方法 | P@1 | kNN@5 | 关键观察 |
|------|-----|-------|---------|
| Euclidean-raw | 83.9% | 85.0% | 无需任何处理，已有不错基准 |
| Euclidean-znorm-perchan | 70.6% | 71.1% | **逐通道归一化反而有害**，振幅信息是关键 |
| Cosine-znorm-perchan | 70.6% | 71.1% | 与上完全相同（z-norm 后余弦=欧氏） |
| DTW-perchan-avg | 79.4% | 78.9% | 每通道独立 DTW 再平均，不如多变量 DTW |
| **DTW-multichan** | **88.9%** | **88.9%** | 全局 z-norm + 多变量 DTW，最强纯 TS 基准 |
| **MTSEmbedder v9** | **90.0%** | 87.8% | 唯一 P@1 超越 DTW 的方法 |
| **MTSEmbedder v18** | 88.9% | 87.8% | P@5、Perfect@5 最优 |

**关键洞察**：LLM 嵌入在 P@1 上（v9=90.0%）超越了最优纯 TS 方法（DTW-multichan=88.9%），但在 kNN@5 分类精度上（87.8% vs 88.9%）略逊一筹，差距仅 2 个样本。这表明 LLM 嵌入更擅长**精确匹配**（top-1），而 DTW 在**邻域结构一致性**（k 个邻居投票）上略有优势。

## 六、已验证的规律（What Works）

1. **TS tokens 位于 prompt 末尾**：因果注意力下，末尾 token 可见所有上下文，是最优的注入位置。
2. **ts_tokens mean pooling**：对 TS token 位置的 hidden states 取均值，避免 last-token recency bias。
3. **全局 z-norm 优于逐通道 z-norm**：通道间的相对幅度比例是 NATOPS 的关键判别特征，不能丢弃。
4. **EMA gallery + memory bank**：稳定负样本分布，避免每 batch 负样本波动。
5. **SoftCLT（C×T flatten + L2 norm）**：以时序距离为软权重，改善了对"相似但不同类"样本的处理。
6. **双视图增强（v18 P12）**：view1 强增强作 query，view2 弱增强（无时间切片）作 EMA gallery，使 gallery 嵌入更稳定，是迄今最有效的单项改进。
7. **Memory bank warmup flush**：warmup 结束时清空 bank，避免低质量嵌入干扰正式训练。

## 七、已探索但失效的方向（What Doesn't Work）

| 方向 | 版本 | 失败原因 |
|------|------|---------|
| CoT 推理（让 LLM 先分析再嵌入） | v10_cot | 推理文本引入噪声，嵌入空间混乱 |
| 实例级 teacher_thought 作 context | v15 | 每个样本的描述各不相同，与对比学习"同类应相似"矛盾 |
| 类级固定 context 描述 | v16 | 模型学会依赖文本 shortcut，未真正学习时序特征 |
| 窗口切片增强 | v14 | 改变峰值位置，NATOPS 中峰值位置是关键特征 |
| Projector 瘦身（128→512） | v13 | 降低了表达能力，未能从减少参数中获益 |
| Cross-Channel Mixer | v13 | 在 v13 中与 Projector 瘦身同时引入，联合退步，单独效果未知 |
| DTW 进行对比软权重（跨视图） | v17 思路 | view1（强增强）与 view2（弱增强）之间的 DTW 距离偏大，软权重趋于均匀，退化为标准 label-based |
| 双视图 + 对角自正样本（hybrid mode） | v17 | 自对比权重 50%，退化为 SimCLR，loss 发散 |
| 通用预训练→NATOPS 微调 | universal v1-3 | 180 样本过拟合，通用预训练未带来迁移增益 |

## 八、当前瓶颈分析

### 8.1 类别 2/3 的持续混淆

所有方法（包括 DTW、LLM 嵌入）均无法完美区分类别 2（All clear）和类别 3（Not clear）：

- 两者均为单臂动作，轨迹相似
- Hit@1 分别仅有 76.7% 和 63.3%
- Perfect@5 = 0%（top-5 中从未出现 5 个同类）

**可能原因**：
(a) 这两类的时序轨迹本身非常接近（传感器噪声 > 类间差异）
(b) 当前 context 描述过于简单，未能捕获区分性语义信息
(c) 180 个训练样本中每类仅 30 个，难以学到精确的决策边界

### 8.2 LLM 带来的增益是否物有所值？

DTW-multichan 在 kNN@5 分类准确率上（88.9%）与 LLM 嵌入（87.8%）相当，甚至略高。LLM 主要在 P@1（top-1 精确匹配）上有优势（v9=90.0% vs DTW=88.9%）。

这提出一个根本问题：**在 NATOPS 这样的小数据集上，LLM 的语义理解是否真正发挥作用，还是只是学了一个过参数化的距离函数？**

从 Perfect@5 的提升（LLM 66.1% vs DTW 63.3%）可以看出，LLM 嵌入确实使同类样本聚合得更紧密，这说明有一定的语义作用，但效果有限。

### 8.3 小数据集的根本限制

NATOPS 训练集 180 个样本，每类 30 个。对比学习通常需要大量样本来学习丰富的负样本关系。当前的 memory bank（90 个槽位）虽然扩大了每 batch 的负样本数，但仍受制于数据集本身的多样性不足。

## 九、未来方向与想法

### 9.1 短期可尝试（低风险）

**P8：ProtoNCE 独立温度**
- 当前 InfoNCE 和 ProtoNCE 共享一个可学习温度，但两者的最优温度不同（InfoNCE 有 108 个 gallery 负样本，ProtoNCE 只有 6 类原型）
- 为 ProtoNCE 加独立 `proto_log_tau`，预期有小幅改进
- 改动量：~10 行代码

**改进 SoftCLT 跨视图权重**
- v18 中 SoftCLT 使用 view1（强增强）计算 DTW 距离，但 view1 有窗口切片，导致距离偏大
- 可以改用 view2（弱增强）的时序计算 SoftCLT 权重，更能反映真实的类内距离关系
- 预期：同类样本的软权重分布更合理

**调整增强强度**
- 当前 view1 使用较强增强（jitter + scale + channel dropout + window slicing），view2 不做 window slicing
- 可以探索不同的增强强度组合，例如 view1 也去掉 channel dropout

### 9.2 中期探索（中等风险）

**更丰富的 Context 描述**（针对类别 2/3）
- v15 用实例级 teacher_thought 失败了，v16 用类级固定描述也失败了
- 一个折中方案：**半结构化 context**，包含类级语义 + 部分量化特征（如"右臂抬起角度约 45 度，持续约 0.8 秒"）
- 关键：保持同类样本 context 的**相似性**（不能每个样本都独特，否则与对比损失矛盾），但又要足够区分不同类

**Hard Negative Mining**
- 当前正样本来自同类（label mode），负样本是随机 gallery + memory bank
- 对于类别 2 和 3，可以主动将它们互相设为 hard negative，施加更大的分离损失
- 实现：在对比损失中对跨视图的难混淆对加权（weight > 1）

**更长的 patch / 更多 TS tokens**
- 当前 patch_size=5，产生 P=6 个 patch，共 144 个 TS tokens（24 通道 × 6 patch）
- 减小 patch_size（如 patch_size=3）可以产生更细粒度的时序表示（P≈8, 192 tokens）
- 权衡：tokens 增多会增加 VRAM 和计算量，以及 LLM 处理 192 个"视觉 token"的难度

**ChannelMixer 单独测试**
- v13 同时引入了 Projector 瘦身和 ChannelMixer，两者联合退步
- 单独在 v12 架构上（保持原始 Projector）测试 ChannelMixer，分离效果

### 9.3 长期方向（高风险，高收益）

**更大数据集验证**
- NATOPS 只有 180 个训练样本，LLM 的优势可能在更大数据集上更明显
- 下一步应在更多 UEA 数据集（已有 UEA_Archive/）上测试泛化能力
- 关键问题：当前系统能否在未见过的时序领域（如 ECG、EEG、工业传感器）保持优势？

**通用嵌入器重试（更大数据集）**
- 之前的通用预训练（universal_mix_v1，1353 样本）→ NATOPS 微调失败，主要因为 NATOPS 太小（过拟合）
- 如果扩展到更大的多数据集混合（数万样本），通用预训练可能真正有效
- 想法：先在大规模数据上预训练一个通用时序嵌入器，然后 zero-shot 或 few-shot 迁移

**替换 LLM backbone**
- 当前用 Qwen3.5-4B（约 4B 参数，LoRA 微调可训练参数 10M）
- 更小的 LLM（如 Qwen3.5-1.5B）是否能在大幅减少计算量的同时保持性能？
- 或者：不用 LLM，改用专用的时序 Transformer（如 PatchTST、TimesNet）+ 对比学习，是更直接的对比

**端到端 DTW 近似**
- 当前 DTW 只在推理时用于混合检索（alpha=0.9 中的 10%），没有参与训练
- 如果用可微分的 DTW（如 soft-DTW）作为额外损失，让嵌入空间与 DTW 距离对齐，可能改善 kNN 精度

## 十、总结

**当前状态**：v18 是最优版本，在 NATOPS 测试集上达到 85.56% 分类准确率、P@1=88.9%、Perfect@5=66.1%，在已探索的所有方向中排名第一。

**最大的未解问题**：类别 2（All clear）和类别 3（Not clear）的混淆，所有方法都在 63-77% 徘徊，是系统的主要瓶颈。

**最有希望的下一步**：
1. P8 独立 ProtoNCE 温度（低风险，短期）
2. 在更多 UEA 数据集上验证泛化能力（中期）
3. 更大规模的通用预训练（长期）

**一个核心反思**：LLM 的引入在 NATOPS 上带来了边际改进（v9 比 DTW-multichan P@1 高 1.1%，v18 的 Perfect@5 比 DTW 高 2.8%），但改进幅度远小于预期。这可能说明：(a) 当前 context 描述不够区分性，LLM 的语义理解未充分发挥；(b) NATOPS 的本质特征是时序形态，而非语义，LLM 的强项（语言推理）与任务错位；(c) 数据量太小，LLM 的表达能力有冗余。这些反思为后续更大规模实验提供了方向。
