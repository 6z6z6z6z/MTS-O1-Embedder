# MTS-O1-Embedder 重构方案：面向多粒度时序检索的基础表征模型

> 基于《时序检索基础模型思考》的目标定义、v9–v18 全部实验数据、系统缺陷分析、以及 Gemini Embedding / BGE-M3 / MOMENT / NV-Embed / TimesFM 等前沿工作的技术参考
>
> 撰写日期：2026-03-27

---

## 一、重构目标

当前 MTS-O1-Embedder 是一个**单粒度、单视角、单数据集**的对比嵌入系统。v9–v18 的 10 个版本迭代证明，在 NATOPS（180 样本）上，4B LLM 系统仅与零参数 DTW 持平（kNN@5: 87.8% vs 88.9%）。

重构的目标是将其转变为一个**多粒度、多视角、跨域通用**的时序检索基础表征模型，能够：

1. **支持多种相似性检索**：形态相似、趋势机制相似、语义相似、任务相关相似
2. **产出多粒度表征**：patch-level 局部特征、sequence-level 全局特征、结构分解特征（趋势/周期/残差）
3. **跨域通用**：统一处理不同采样率、量纲、通道数、序列长度的时序数据
4. **灵活部署**：通过 Matryoshka 表征支持 256/512/1024 维弹性嵌入

## 二、当前系统评估与核心矛盾

### 2.1 已证实的结构性缺陷

| 缺陷 | 证据 | 根因 |
|------|------|------|
| TS Encoder 是瓶颈 | 绕过 LLM 仅 63.9% P@1，DTW 达 88.9% | stem stride + patch_size 过度压缩；通道独立丢失交互 |
| LLM 价值未释放 | CoT/teacher_thought/类描述均退步；context-only 最优 | 180 样本不足以学习语义对齐；LLM 被当作注意力融合器 |
| 评估协议偏差 | 单数据集 1-2% 差异不具统计显著性 | NATOPS 180 样本 LOO 高方差 |
| Class 2/3 硬天花板 | Perfect@5=0%，所有方法 Hit@1 < 77% | 传感器分辨率不足，类间差异小于噪声 |

### 2.2 核心矛盾

> **用了 4B 参数的 LLM，但表现仅与零参数 DTW 持平。**

这不是调参问题，而是系统**定位**问题。当前系统试图同时做两件矛盾的事：
- 让 CNN 做特征压缩（但压缩过程丢失了判别信息）
- 让 LLM 做语义推理（但所有样本共享同一 context，LLM 从文本侧获得的信息量为零）

重构需要从根本上重新定义每个组件的角色。

## 三、相关工作与技术参考

### 3.1 Gemini Embedding（Google, 2025）

**核心思路**：从预训练生成式 LLM（Gemini）出发构建通用嵌入模型。

- **架构**：基于 Gemini 基础模型，继承其多模态理解能力，产出 3072 维嵌入
- **任务指令机制**：通过 task type（8 种：retrieval、classification、clustering、similarity 等）条件化嵌入，不同任务产出不同的嵌入空间偏好
- **Matryoshka 训练**：支持嵌入截断（3072→768→256），前 N 维包含最显著的语义信息
- **合成数据增强**：通过合成数据扩展 retrieval 和 classification 训练集
- **成果**：MMTEB 排行榜第一，250+ 语言，比第二名高 5.09 分

**对本项目的启示**：
- 从预训练 LLM 出发做嵌入是可行的，但需要足够的训练数据和任务多样性
- task-instruction 机制可以直接借用——让同一个模型根据指令产出"形态嵌入"或"语义嵌入"
- Matryoshka 训练应该标配，支持部署灵活性

### 3.2 BGE-M3（BAAI, 2024）

**核心思路**：单模型同时支持三种检索范式。

- **Dense Retrieval**：[CLS] 向量 → 固定长度嵌入 → 余弦相似度
- **Sparse Retrieval**：token-level 线性投影 + ReLU → 加权词袋 → 词法匹配
- **Multi-vector Retrieval（Late Interaction）**：token 嵌入投影到辅助空间 → ColBERT 风格 MaxSim
- **Self-Knowledge Distillation**：三种检索功能的 relevance scores 互为教师信号
- **多粒度输入**：支持 sentence 到 8192 tokens 长文档

**对本项目的启示**：
- **三头检索架构可直接迁移到时序**：
  - Dense = sequence-level 全局嵌入（余弦）
  - Multi-vector = patch-level late interaction（对应时序局部模式匹配）
  - Sparse = 可替换为结构特征向量（趋势/周期/振幅描述符）
- Self-Knowledge Distillation 可用于对齐不同粒度的检索信号

### 3.3 NV-Embed（NVIDIA, ICLR 2025 Spotlight）

**核心思路**：将 decoder-only LLM 转为通用嵌入模型的最佳实践。

- **双向注意力**：移除 decoder 的 causal mask，改用 bidirectional attention，在所有 pooling 方式上显著优于 causal mask
- **Latent Attention Pooling**：用可学习的 latent vectors 做 cross-attention，替代 mean pooling 或 last-token，持续提升检索和下游任务精度
- **两阶段对比指令微调**：
  1. Stage 1：在 retrieval 数据集上做对比训练（in-batch negatives + hard negatives）
  2. Stage 2：扩展到 non-retrieval 任务（分类、聚类、语义相似度）

**对本项目的启示**：
- **bidirectional attention 是关键改进**——当前系统用 causal mask，TS tokens 在末尾虽然能看到所有文本，但文本 tokens 看不到 TS tokens。改为双向后，文本和时序可以完全交互
- **Latent Attention Pooling 应替代当前 ts_tokens mean pooling**——可学习的 latent queries 能自适应地聚合重要信息
- 两阶段训练的第一阶段对应当前的对比学习，第二阶段可扩展到分类/聚类任务

### 3.4 MOMENT（CMU, ICML 2024）

**核心思路**：时序基础模型的 patch tokenization 和预训练方法。

- **Patch Tokenization**：将时序切分为固定长度 patch，每个 patch 视为一个 token，显著提升推理速度且不约束预测长度
- **Masked Reconstruction 预训练**：随机 mask 部分 patch，训练模型重建（类似 BERT 的 MLM）
- **Time Series Pile**：构建大规模多样化公开时序数据集合，系统解决跨域预训练难题
- **通用任务**：预训练后支持 forecasting、classification、anomaly detection、imputation

**对本项目的启示**：
- Patch tokenization 的思路与当前 CNN patching 一致，但 MOMENT 使用更简洁的**线性 patch 投影**（无 CNN 特征提取），信息损失更小
- **Masked reconstruction 应作为预训练阶段的辅助目标**——帮助模型理解时序的局部结构
- Time Series Pile 的数据组织方式可参考——按 domain、频率、长度分层组织

### 3.5 TimesFM（Google, ICML 2024）

**核心思路**：decoder-only 时序基础模型。

- **架构**：stacked transformer layers + patch tokenization，200M 参数
- **Input/Output Patch 不对称**：output_patch_size > input_patch_size，加速推理并提升精度
- **100B 真实数据预训练**：在 100 billion 真实时间点上预训练，覆盖多个领域和时间粒度
- **Zero-shot 泛化**：在未见过的数据集上接近有监督 SOTA

**对本项目的启示**：
- 200M 参数 + 大规模数据 > 4B 参数 + 小数据；**数据规模比模型规模更重要**
- Decoder-only + patching 的架构设计可以工作，但需要足够的预训练数据
- Output patch 不对称的思想可以迁移：嵌入头可以在更粗粒度上生成全局表征

### 3.6 TS2Vec / SoftCLT

- **TS2Vec**：层次化对比学习，同时在 timestamp-level 和 instance-level 做对比，捕获多尺度时序结构
- **SoftCLT（ICLR 2024）**：用时序距离做软正样本权重，替代硬 0/1 标签

**对本项目的启示**：当前系统已使用 SoftCLT（P7），但只在 instance-level 做对比。可扩展到 **patch-level 对比**（同一时序的对齐 patch 互为正样本）和 **trend-level 对比**（分解后的趋势项做对比）。

### 3.7 Soft-DTW（可微 DTW 损失）

- **原理**：用 softmin 替代 DTW 中的 min 操作，使整个对齐路径可微
- **复杂度**：O(T²) 时间和空间，与标准 DTW 相同
- **优势**：可作为端到端训练损失，让嵌入空间与 DTW 距离结构对齐

**对本项目的启示**：
- 当前 DTW 仅用于推理时的混合检索（10% 权重），不参与训练
- **Soft-DTW 应作为辅助训练损失**：让 patch-level 表征的距离与 DTW 对齐距离一致，将 DTW 的结构对齐能力内化到模型中

### 4.2 与当前架构的对比

| 维度 | 当前架构 (v18) | 重构架构 |
|------|---------------|---------|
| **TS 编码** | CNN (stem stride + ResBlock + Conv patch) | 线性 Patch Embedding + TC-Former |
| **信息损失** | 严重（63.9% P@1 绕过 LLM） | 最小化（无 stride 下采样，直接线性投影） |
| **通道交互** | Channel-Independent（可选 ChannelMixer） | 内置交替 Temporal-Channel Attention |
| **LLM 注意力** | Causal mask（TS 只能看前面的文本） | Bidirectional（TS 与文本完全交互） |
| **Pooling** | ts_tokens mean pooling | Latent Attention Pooling（可学习聚合） |
| **LLM 规模** | Qwen3.5-4B（语义未利用） | Qwen2.5-1.5B（更小但更有效） |
| **检索输出** | 单一 1024 维 dense embedding | 三头：Dense + Multi-Vector + Structural |
| **嵌入维度** | 固定 1024 维 | MRL 支持 256/512/1024/2048 弹性维度 |
| **任务条件** | 固定 context 字符串 | task_instruction 条件化不同检索类型 |
| **跨域能力** | NATOPS 单数据集 | RevIN + Adaptive Patching + 多数据集预训练 |

### 4.3 各组件详细设计

#### 4.3.1 自适应输入标准化（Adaptive Input Normalization）

**问题**：不同领域的时序在量纲、采样率、噪声模式上差异巨大。当前系统无任何标准化（全局 z-norm 在 NATOPS 上最优，但逐通道 z-norm 退步 18%）。

**方案**：RevIN（Reversible Instance Normalization），在模型入口处标准化，在输出处可恢复：

```python
class RevIN(nn.Module):
    """Reversible Instance Normalization (Kim et al., 2021)"""
    def __init__(self, num_channels, affine=True):
        super().__init__()
        self.affine = affine
        if affine:
            self.gamma = nn.Parameter(torch.ones(1, num_channels, 1))
            self.beta = nn.Parameter(torch.zeros(1, num_channels, 1))

    def forward(self, x, mode='norm'):
        # x: [B, C, T]
        if mode == 'norm':
            self._mean = x.mean(dim=-1, keepdim=True).detach()
            self._std = (x.std(dim=-1, keepdim=True) + 1e-5).detach()
            x = (x - self._mean) / self._std
            if self.affine:
                x = x * self.gamma + self.beta
        elif mode == 'denorm':
            if self.affine:
                x = (x - self.beta) / self.gamma
            x = x * self._std + self._mean
        return x
```

**关键决策**：RevIN 对每个样本独立计算统计量，不同于全局 z-norm。前者保留了跨通道的相对幅度比例（NATOPS 的关键特征），同时消除绝对尺度差异。affine 参数允许模型学习最优的标准化偏移。

**自适应 Patch Size**：

```python
def get_patch_size(seq_len: int) -> int:
    if seq_len <= 64:
        return 4       # 短序列: 细粒度 patch
    elif seq_len <= 256:
        return 8       # 中等序列
    elif seq_len <= 1024:
        return 16      # 长序列
    else:
        return 32      # 超长序列
```

对于 NATOPS（T=51）：patch_size=4，P=13（ceil(51/4)，末尾 padding），24×13=312 个 patch tokens。比当前的 144 tokens 更多，但每个 patch 仅覆盖 4 个时间步，分辨率更高。

#### 4.3.2 多粒度时序编码器（Multi-Granularity TS Encoder）

**设计哲学**：当前 CNN encoder 的问题是"过度压缩"——stem stride + ResBlock + Conv patch 在 51 步上只保留 6 个 patch 位置。MOMENT 和 TimesFM 的成功表明，**简单的线性 patch 投影 + Transformer** 在大规模数据上效果更好。

**Patch Embedding Layer**：

```python
class PatchEmbedding(nn.Module):
    def __init__(self, patch_size, d_model, max_channels=64, max_patches=128, dropout=0.1):
        super().__init__()
        self.patch_size = patch_size
        self.proj = nn.Linear(patch_size, d_model)
        self.norm = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)
        # 可学习的位置和通道嵌入
        self.pos_embed = nn.Embedding(max_patches, d_model)
        self.chan_embed = nn.Embedding(max_channels, d_model)

    def forward(self, x, num_channels):
        # x: [B, C, T] (已 RevIN 标准化)
        B, C, T = x.shape
        # 通道独立 unfold
        x = x.reshape(B * C, T)                        # [B*C, T]
        # padding to multiple of patch_size
        pad_len = (self.patch_size - T % self.patch_size) % self.patch_size
        if pad_len > 0:
            x = F.pad(x, (0, pad_len))
        P = x.shape[-1] // self.patch_size
        x = x.reshape(B * C, P, self.patch_size)        # [B*C, P, p]
        x = self.proj(x)                                # [B*C, P, d_model]
        x = self.norm(x)
        x = self.dropout(x)
        # 添加位置和通道嵌入
        pos_ids = torch.arange(P, device=x.device)
        x = x + self.pos_embed(pos_ids).unsqueeze(0)     # broadcast over B*C
        x = x.reshape(B, C, P, -1)                      # [B, C, P, d]
        chan_ids = torch.arange(C, device=x.device)
        x = x + self.chan_embed(chan_ids).reshape(1, C, 1, -1)
        x = x.reshape(B, C * P, -1)                     # [B, C*P, d_model]
        return x, P
```

**为什么不用 CNN**：
1. 线性投影是"无损"的——每个 patch 的完整 p 维信息直接映射到 d_model，无 stride 压缩
2. 位置和通道嵌入让 Transformer 自行学习什么位置/通道重要
3. 参数量极小：`patch_size × d_model + max_patches × d_model + max_channels × d_model`

**TC-Former（Temporal-Channel Transformer）**：

```python
class TCFormerLayer(nn.Module):
    """交替 Temporal 和 Channel Self-Attention"""
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        super().__init__()
        # Temporal attention (across patches within each channel)
        self.temporal_attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.temporal_norm = nn.LayerNorm(d_model)
        # Channel attention (across channels at each patch position)
        self.channel_attn = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.channel_norm = nn.LayerNorm(d_model)
        # FFN
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Dropout(dropout),
            nn.Linear(d_ff, d_model), nn.Dropout(dropout)
        )
        self.ffn_norm = nn.LayerNorm(d_model)

    def forward(self, x, C, P):
        B = x.shape[0]
        # (a) Temporal self-attention: [B*C, P, d]
        x_t = x.reshape(B * C, P, -1)
        x_t = self.temporal_norm(x_t)
        attn_out, _ = self.temporal_attn(x_t, x_t, x_t)
        x = x + attn_out.reshape(B, C * P, -1)
        # (b) Channel self-attention: [B*P, C, d]
        x_c = x.reshape(B, C, P, -1).permute(0, 2, 1, 3).reshape(B * P, C, -1)
        x_c = self.channel_norm(x_c)
        attn_out, _ = self.channel_attn(x_c, x_c, x_c)
        x = x + attn_out.reshape(B, P, C, -1).permute(0, 2, 1, 3).reshape(B, C * P, -1)
        # (c) FFN
        x = x + self.ffn(self.ffn_norm(x))
        return x
```

**与当前 ChannelMixer 的区别**：
- ChannelMixer 是 encoder 后的单层 add-on；TC-Former 是 L 层交替的核心编码器
- ChannelMixer 只做 channel attention；TC-Former 同时做 temporal + channel attention
- TC-Former 的 temporal attention 替代了 CNN 的 ResBlock，用注意力而非卷积捕获时间依赖

**推荐配置**：

| 规模 | d_model | n_heads | d_ff | L 层 | 参数量 |
|------|---------|---------|------|------|--------|
| Small | 128 | 4 | 512 | 4 | ~2M |
| Base | 256 | 8 | 1024 | 6 | ~12M |
| Large | 512 | 8 | 2048 | 8 | ~45M |

NATOPS 实验建议从 Small 开始（与当前 CNN hidden_dim=128 对齐），验证编码质量后再扩展。

#### 4.3.3 LLM 语义增强层

**当前问题**：LLM 被当作"盲目注意力融合器"，原因有三：
1. Causal mask 限制了 TS↔text 交互
2. 所有样本共享同一 context，LLM 从文本获得零信息
3. 4B 参数 + 180 样本 = 严重过参数化

**重构方案**：

**a) 双向注意力（Bidirectional Attention）**

参考 NV-Embed，移除 causal mask：

```python
# 当前: causal attention (LLM 默认)
# 重构: 修改 attention mask 为 bidirectional
attention_mask = torch.ones(batch_size, seq_len, seq_len, device=device)
# 所有 token 可以看到所有 token，包括 TS tokens 和 text tokens 的双向交互
```

这让文本 tokens（如通道描述）能够"看到" TS tokens，从而真正做到语义-信号交互，而非当前的单向信息流。

**b) 结构化通道描述（Structured Channel Description）**

替代当前的通用 context（对所有样本相同），为每个数据集提供**结构化通道描述**：

```
task_instruction: "Retrieve time series with similar motion patterns."

channel_description:
"Channel layout: 8 body locations × 3 axes (X/Y/Z).
 Ch 1-3: Right hand (XYZ). Ch 4-6: Right elbow (XYZ).
 Ch 7-9: Right shoulder (XYZ). Ch 10-12: Left hand (XYZ).
 Ch 13-15: Left elbow (XYZ). Ch 16-18: Left shoulder (XYZ).
 Ch 19-21: Head (XYZ). Ch 22-24: Torso (XYZ)."
```

关键区别：
- v15/v16 失败是因为给每个样本/类不同的**描述性文本**，导致对比学习与文本 shortcut 矛盾
- 这里的通道描述对**所有样本完全相同**，不改变对比学习的同类相似假设
- 但它给 LLM 提供了**物理语义**——知道 Ch1-3 是右手、Ch12-14 是左手，就能理解"对称运动"的含义

**c) Task Instruction 条件化**

参考 Gemini Embedding 的 task type 机制，为不同检索需求提供不同指令：

```python
TASK_INSTRUCTIONS = {
    "shape_retrieval": "Retrieve time series with similar waveform shapes and local patterns.",
    "trend_retrieval": "Retrieve time series with similar dynamic evolution patterns (trends, cycles, regime changes).",
    "semantic_retrieval": "Retrieve time series from similar scenarios or mechanisms, regardless of scale.",
    "classification": "Classify this time series into the most appropriate category.",
    "clustering": "Generate an embedding suitable for grouping similar time series together.",
}
```

训练时随机采样 task instruction（多任务训练），推理时根据用户需求选择。

**d) Latent Attention Pooling**

替代当前 ts_tokens mean pooling：

```python
class LatentAttentionPooling(nn.Module):
    def __init__(self, d_model, num_latents=8, n_heads=8):
        super().__init__()
        self.latent_queries = nn.Parameter(torch.randn(num_latents, d_model) * 0.02)
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm = nn.LayerNorm(d_model)

    def forward(self, hidden_states, attention_mask=None):
        B = hidden_states.shape[0]
        queries = self.latent_queries.unsqueeze(0).expand(B, -1, -1)  # [B, K, d]
        # Cross-attention: latent queries attend to all hidden states
        pooled, _ = self.cross_attn(
            query=queries,
            key=hidden_states,
            value=hidden_states,
            key_padding_mask=(~attention_mask.bool()) if attention_mask is not None else None
        )
        pooled = self.norm(pooled)           # [B, K, d]
        return pooled.mean(dim=1)            # [B, d]  全局表征
```

**为什么优于 mean pooling**：
- Mean pooling 平等对待所有 token，包括 padding 和低信息 token
- Latent Attention 通过可学习的 queries 自适应地关注最重要的 token
- NV-Embed 实验证明 Latent Attention 在所有 MTEB 任务上持续优于 mean/last pooling

**e) 降低 LLM 规模**

从 Qwen3.5-4B 降至 **Qwen2.5-1.5B**：
- 减少 ~2.5B 参数，降低 VRAM 需求（~16GB → ~6GB）
- 如果 v9 的缺陷分析正确（LLM 语义未利用），1.5B 应该性能持平
- 更小的模型 + bidirectional attention + structured context 可能优于更大的模型 + causal mask + empty context
- 若 1.5B 仍然过大，可进一步降至 0.5B（如 README 中提及的 Qwen2.5-0.5B）

#### 4.3.4 多头检索输出

参考 BGE-M3 的三检索范式，设计三个检索头：

**a) Dense Head（全局语义检索）**

```python
class DenseHead(nn.Module):
    """MRL-aware dense embedding head"""
    def __init__(self, input_dim, output_dims=[256, 512, 1024]):
        super().__init__()
        self.max_dim = max(output_dims)
        self.output_dims = sorted(output_dims)
        self.projection = nn.Linear(input_dim, self.max_dim)
        self.norm = nn.LayerNorm(self.max_dim)

    def forward(self, x):
        x = self.projection(x)
        x = self.norm(x)
        x = F.normalize(x, dim=-1)
        return x  # [B, max_dim], 可截断到任意 output_dims
```

MRL 训练：在每个 output_dim 上分别计算对比损失，加权求和：

```python
def matryoshka_loss(embeddings, labels, dims=[256, 512, 1024]):
    total = 0
    for d in dims:
        trunc = F.normalize(embeddings[:, :d], dim=-1)
        total += infonce_loss(trunc, labels)
    return total / len(dims)
```

**b) Multi-Vector Head（局部形态匹配）**

```python
class MultiVectorHead(nn.Module):
    """ColBERT-style late interaction for patch-level matching"""
    def __init__(self, d_model, d_mv=64):
        super().__init__()
        self.proj = nn.Linear(d_model, d_mv)

    def forward(self, patch_tokens):
        # patch_tokens: [B, C*P, d_model]
        mv = self.proj(patch_tokens)        # [B, C*P, d_mv]
        mv = F.normalize(mv, dim=-1)
        return mv

    @staticmethod
    def maxsim(q_mv, g_mv):
        # q_mv: [Nq, n_patches_q, d_mv]
        # g_mv: [Ng, n_patches_g, d_mv]
        # → sim_matrix: [Nq, Ng]
        # 对 query 的每个 patch，找到 gallery 中最相似的 patch，取均值
        sim = torch.einsum('qid,gjd->qgij', q_mv, g_mv)  # [Nq, Ng, Pq, Pg]
        max_sim = sim.max(dim=-1).values                    # [Nq, Ng, Pq]
        return max_sim.mean(dim=-1)                          # [Nq, Ng]
```

**用途**：当两条时序的全局嵌入相似但局部模式不同时（如趋势相同但局部 spike 不同），Multi-Vector 可以提供更细粒度的匹配信号。

**c) Structural Head（趋势/周期/统计特征）**

```python
class StructuralHead(nn.Module):
    """基于时序统计和频谱特征的结构描述符"""
    def __init__(self, num_channels, d_struct=128):
        super().__init__()
        # 输入特征: RevIN stats(mean, std per channel) + spectral features
        # 每通道: mean, std, skew, kurtosis, dominant_freq, spectral_entropy = 6
        feat_dim = num_channels * 6
        self.mlp = nn.Sequential(
            nn.Linear(feat_dim, d_struct * 2),
            nn.GELU(),
            nn.Linear(d_struct * 2, d_struct),
            nn.LayerNorm(d_struct),
        )

    def forward(self, ts_raw, revin_stats):
        # 从原始时序提取统计 + 频谱特征
        feats = self._extract_features(ts_raw, revin_stats)
        return F.normalize(self.mlp(feats), dim=-1)

    def _extract_features(self, ts, stats):
        B, C, T = ts.shape
        mean, std = stats['mean'], stats['std']              # [B, C]
        skew = ((ts - mean.unsqueeze(-1)) ** 3).mean(-1) / (std ** 3 + 1e-8)
        kurtosis = ((ts - mean.unsqueeze(-1)) ** 4).mean(-1) / (std ** 4 + 1e-8)
        # FFT 特征
        fft = torch.fft.rfft(ts, dim=-1)
        power = (fft.real ** 2 + fft.imag ** 2)
        dominant_freq = power.argmax(dim=-1).float() / T     # [B, C]
        power_norm = power / (power.sum(dim=-1, keepdim=True) + 1e-8)
        spectral_entropy = -(power_norm * (power_norm + 1e-8).log()).sum(dim=-1)
        feats = torch.cat([mean, std, skew, kurtosis, dominant_freq, spectral_entropy], dim=-1)
        return feats
```

**用途**：捕获趋势/周期/分布层面的相似性。两条来自不同领域、数值尺度不同的时序，如果呈现相似的"上升后震荡"模式，Structural Head 能捕获这种趋势机制相似性。

#### 4.3.5 混合检索融合

```python
class HybridMultiGranularityRetriever:
    """三头融合检索"""
    def __init__(self, alpha_dense=0.6, alpha_mv=0.3, alpha_struct=0.1):
        self.alpha_dense = alpha_dense
        self.alpha_mv = alpha_mv
        self.alpha_struct = alpha_struct

    def search(self, query, gallery, k=5):
        # Dense similarity
        sim_dense = cosine_similarity(query.dense_emb, gallery.dense_emb)
        # Multi-vector similarity (MaxSim)
        sim_mv = MultiVectorHead.maxsim(query.mv_tokens, gallery.mv_tokens)
        # Structural similarity
        sim_struct = cosine_similarity(query.struct_emb, gallery.struct_emb)
        # Normalize and fuse
        final_sim = (self.alpha_dense * normalize(sim_dense) +
                     self.alpha_mv * normalize(sim_mv) +
                     self.alpha_struct * normalize(sim_struct))
        return final_sim.topk(k)
```

融合权重可通过 task instruction 动态调整：
- shape_retrieval: alpha_mv=0.6, alpha_dense=0.3, alpha_struct=0.1
- trend_retrieval: alpha_struct=0.5, alpha_dense=0.4, alpha_mv=0.1
- semantic_retrieval: alpha_dense=0.7, alpha_mv=0.2, alpha_struct=0.1

### 5.3 掩码 Patch 重建（Masked Patch Reconstruction）

参考 MOMENT，随机 mask 部分 patch tokens，训练 encoder 重建：

```python
class MaskedPatchReconstruction:
    def __init__(self, mask_ratio=0.4, d_model=128, patch_size=4):
        self.mask_ratio = mask_ratio
        self.decoder = nn.Linear(d_model, patch_size)  # 轻量 decoder

    def __call__(self, patch_tokens, original_patches):
        B, N, d = patch_tokens.shape
        num_mask = int(N * self.mask_ratio)
        # 随机选择 mask 位置
        mask_ids = torch.randperm(N)[:num_mask]
        # 用可学习的 mask token 替换
        masked_tokens = patch_tokens.clone()
        masked_tokens[:, mask_ids] = self.mask_token
        # 通过 encoder 重建
        reconstructed = self.decoder(masked_tokens[:, mask_ids])
        target = original_patches[:, mask_ids]
        return F.mse_loss(reconstructed, target)
```

**作用**：迫使 encoder 理解 patch 之间的时间依赖关系，而非仅依赖对比信号。对于跨域预训练（无标签时），这是唯一的自监督信号源。

### 5.4 Soft-DTW 结构对齐损失

将 DTW 的结构对齐能力内化到 patch-level 表征中：

```python
from tslearn.metrics import SoftDTWLossPyTorch

class SoftDTWAlignmentLoss:
    def __init__(self, gamma=0.1):
        self.gamma = gamma

    def __call__(self, patch_tokens_q, patch_tokens_g, labels):
        # 对同类样本：patch 序列的 Soft-DTW 距离应该小
        # 对异类样本：Soft-DTW 距离应该大
        B = patch_tokens_q.shape[0]
        loss = 0
        count = 0
        for i in range(B):
            for j in range(B):
                sdtw = soft_dtw(patch_tokens_q[i], patch_tokens_g[j], gamma=self.gamma)
                if labels[i] == labels[j]:
                    loss += sdtw           # 正样本：最小化距离
                else:
                    loss += max(0, margin - sdtw)  # 负样本：拉开距离
                count += 1
        return loss / count
```

**注意**：Soft-DTW 的 O(P²) 复杂度在 patch 数量不大时（P=13 for NATOPS）完全可接受。对于长序列（P>64），可使用 Fast Soft-DTW 近似。

### 5.5 Matryoshka 多尺度损失

```python
def matryoshka_contrastive_loss(embeddings, labels, dims=[256, 512, 1024]):
    """在嵌套的子维度上分别计算对比损失"""
    weights = {256: 0.2, 512: 0.3, 1024: 0.5}  # 越大维度权重越高
    total = 0
    for d in dims:
        trunc = F.normalize(embeddings[:, :d], dim=-1)
        total += weights[d] * infonce_loss(trunc, labels)
    return total
```

### 5.6 Self-Knowledge Distillation（三头互蒸馏）

参考 BGE-M3，三个检索头的 relevance scores 互为教师信号：

```python
def self_distillation_loss(sim_dense, sim_mv, sim_struct, temperature=1.0):
    """三头 score 的一致性约束"""
    # 将三个 score 合并为 teacher signal
    teacher = (sim_dense + sim_mv + sim_struct) / 3
    teacher = F.softmax(teacher / temperature, dim=-1).detach()
    # 每个头向 teacher 学习
    loss_d = F.kl_div(F.log_softmax(sim_dense / temperature, dim=-1), teacher, reduction='batchmean')
    loss_m = F.kl_div(F.log_softmax(sim_mv / temperature, dim=-1), teacher, reduction='batchmean')
    loss_s = F.kl_div(F.log_softmax(sim_struct / temperature, dim=-1), teacher, reduction='batchmean')
    return (loss_d + loss_m + loss_s) / 3
```

在特定数据集上微调（如 NATOPS），使用全部损失目标：

```python
finetune_config = {
    "objectives": ["contrastive", "soft_dtw", "matryoshka", "self_distillation"],
    "epochs": 40,
    "lr": 1e-4,
    "dataset": "natops",
    "llm_enabled": True,       # 此阶段加入 LLM
    "bidirectional": True,
    "channel_descriptions": True,
}
```

### 6.3 跨域关键技术

**a) RevIN 处理量纲差异**：每个样本独立标准化，消除绝对尺度差异

**b) Adaptive Patching 处理序列长度差异**：根据 T 自动选择 patch_size，确保 patch 数量在合理范围（8-64）

**c) Channel Padding 处理通道数差异**：padding 到 batch 内最大通道数，channel attention 用 mask 忽略 padding 通道

**d) Dataset-specific Prompt**：每个数据集的 channel_description 不同，但同一数据集内所有样本共享（避免 v15/v16 的文本 shortcut 问题）

## 七、检索评估框架

### 7.1 多类型检索基准

参考原始思考文档中的四类检索任务定义：

**Task 1: 形态检索（Shape Retrieval）**

> 给定一段查询序列，检索形态最相近的历史片段。

- **Ground Truth**：DTW 距离最近的 k 个样本（shape-based nearest neighbors）
- **评估指标**：Recall@k（检索结果与 DTW 近邻的重叠率）、NDCG@k
- **意义**：测试 Multi-Vector Head 的局部模式匹配能力

**Task 2: 趋势机制检索（Trend/Mechanism Retrieval）**

> 给定一个当前场景，检索未来演化模式最接近的 case。

- **Ground Truth**：结构分解（STL）后趋势项最相似的样本
- **评估指标**：趋势项余弦相似度的 Recall@k
- **意义**：测试 Structural Head 的趋势/周期特征捕获能力

**Task 3: 语义检索（Semantic Retrieval）**

> 给定带上下文的多变量序列，检索相同机制下的跨域样本。

- **Ground Truth**：同类标签（label-based）
- **评估指标**：P@k、Hit@k、Perfect@k、kNN 分类准确率、Macro-F1
- **意义**：当前系统的主要评估方式，测试 Dense Head 的语义对齐能力

**Task 4: 检索增强预测（Retrieval-Augmented Forecasting，中长期目标）**

> 验证检索到的记忆是否真的提升预测质量。

- **方法**：用检索到的 k 个最相似历史序列的未来走势，辅助当前序列的预测
- **评估指标**：MAE/MSE（有检索 vs 无检索的预测误差对比）
- **意义**：时序检索的最终实用价值验证

### 7.2 多数据集评估协议

```python
EVALUATION_DATASETS = {
    "in_domain": ["NATOPS"],                                          # 训练时见过
    "near_domain": ["BasicMotions", "RacketSports", "Cricket"],       # 同类传感器
    "out_domain": ["Epilepsy", "ArticularyWordRecognition", "Libras"],# 不同领域
}

METRICS = {
    "semantic": ["P@1", "P@3", "P@5", "Hit@5", "Perfect@5", "kNN_Acc", "Macro_F1"],
    "shape": ["Recall@5_vs_DTW", "NDCG@5_vs_DTW"],
    "structural": ["Recall@5_vs_trend", "Trend_cosine_corr"],
}
```

每个架构改进必须在**至少 3 个数据集**上报告结果，单数据集 1-2% 波动不作为改进证据。

### 7.3 统计显著性要求

- **5-fold Stratified Cross-Validation**：报告 mean ± std
- **Bootstrap 95% 置信区间**：对 P@k 和 kNN Acc 计算 1000 次 bootstrap 重采样的 CI
- **McNemar 检验**：对比两个模型时，检验预测差异是否显著（p < 0.05）

```python
def bootstrap_ci(predictions, labels, metric_fn, n_bootstrap=1000, ci=0.95):
    scores = []
    n = len(predictions)
    for _ in range(n_bootstrap):
        idx = np.random.choice(n, n, replace=True)
        score = metric_fn(predictions[idx], labels[idx])
        scores.append(score)
    lower = np.percentile(scores, (1 - ci) / 2 * 100)
    upper = np.percentile(scores, (1 + ci) / 2 * 100)
    return np.mean(scores), lower, upper
```

## 八、正负样本构造策略

原始思考文档指出："时间序列什么算正样本、什么算负样本，并不总是明确。"

### 8.1 多层级正样本定义

| 层级 | 正样本定义 | 负样本定义 | 用途 |
|------|-----------|-----------|------|
| **Label-based** | 同类标签 | 不同标签 | 语义检索（当前方式） |
| **Augmentation-based** | 同一样本的不同增强视图 | 不同样本 | 自监督对比（无标签时） |
| **DTW-based（软权重）** | SoftCLT: w=exp(-DTW_dist/σ) | 距离大的样本权重低 | 形态相似性（当前 SoftCLT） |
| **Trend-based** | STL 趋势项相似的样本 | 趋势项差异大的样本 | 趋势机制相似性 |
| **Cross-domain** | 不同数据集但标签语义相关的样本 | 语义无关的样本 | 跨域泛化 |

### 8.2 Hard Negative Mining

针对当前 Class 2/3 混淆的硬性天花板：

```python
class HardNegativeMiner:
    """在 batch 内识别困难负样本并加权"""
    def __init__(self, hard_weight=2.0, threshold=0.8):
        self.hard_weight = hard_weight
        self.threshold = threshold

    def get_weights(self, sim_matrix, labels):
        # 找到高相似度但不同类的对 (hard negatives)
        neg_mask = labels.unsqueeze(0) != labels.unsqueeze(1)
        hard_neg_mask = neg_mask & (sim_matrix > self.threshold)
        # 对 hard negatives 给更高权重
        weights = torch.ones_like(sim_matrix)
        weights[hard_neg_mask] = self.hard_weight
        return weights
```

### 8.3 动态正样本挖掘

利用模型自身的嵌入空间动态发现正样本（semi-supervised）：

```python
def mine_positive_pairs(embeddings, labels, top_k=3, min_sim=0.9):
    """在无标签数据中挖掘高置信正样本对"""
    sim = cosine_similarity_matrix(embeddings)
    # 对每个样本，找到 top-k 最相似且不在同一 batch 位置的样本
    for i in range(len(embeddings)):
        topk_idx = sim[i].topk(top_k + 1).indices[1:]  # 排除自身
        for j in topk_idx:
            if sim[i, j] > min_sim:
                # 高置信正样本对，可用于下一轮训练
                yield (i, j.item(), sim[i, j].item())
```

## 十二、总结

本重构方案的核心思路可以概括为四个转变：

1. **编码器从"压缩器"变为"传递器"**：线性 Patch Embedding + TC-Former 替代 CNN，最小化信息损失，同时通过交替 Temporal-Channel Attention 捕获时空交互
2. **LLM 从"盲目融合器"变为"语义增强器"**：bidirectional attention + structured channel descriptions + task instructions，让 LLM 真正理解时序数据的物理语义
3. **检索从"单一相似度"变为"多粒度匹配"**：Dense + Multi-Vector + Structural 三头检索，支持形态/趋势/语义多种相似性需求
4. **评估从"单数据集调参"变为"多数据集验证"**：9+ 数据集、4 类检索任务、5-fold CV + bootstrap CI 的严格评估框架

整个重构预计 14 周，分 4 个 Phase 渐进实施。每个 Phase 有明确的里程碑和验证标准，失败时可以回退到上一个 Phase 的稳定状态。
