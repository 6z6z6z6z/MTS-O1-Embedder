## 问题汇总与改进计划

### 🔴 P1：Memory Bank 冷启动（loss spike 根因）

**问题：** warmup 结束（epoch=warmup_epochs）时，memory bank 里全是前几个 epoch 低质量嵌入，突然加入 InfoNCE 分母使负样本数从 18 增至 108，loss 数值跳升约 2×（1.7→3.8），引起训练不稳定。

**修复：** warmup 结束时 flush bank（`self.memory_bank.reset()`），让 bank 从干净状态重新积累。

**文件：** `mts_agent/engine/trainer.py`
**状态：** ✅ 已实施（v12，trainer.py `EmbeddingMemoryBank.reset()`，epoch==warmup_epochs 触发 flush）

---

### 🔴 P2：TS tokens 位于序列末尾（因果注意力优化）

**问题：** 当前 prompt 结构：
```
[系统指令][Context 文本][TS tokens(144)][Task: Analyze patterns.]
```
因果掩码下，TS tokens 看不到后面的任务指令；研究表明中间位置 visual tokens 受注意力饥渴（attention starvation）影响最严重。

**修复：** 将 TS tokens 移到 prompt 最末尾（retrieval 场景）：
```
[系统指令][Context 文本][Task: Analyze patterns.][TS tokens(144)]
```
最后一个 TS token 能看到所有文本，作为 pooling 点更完整。

**文件：** `mts_agent/data/prompt_templates.py`, `prompt_builder.py`
**参考：** NV-Embed (EOS 位于末尾), HTP (首/尾 token 获得最多注意力)
**状态：** ✅ 已实施（v12，prompt_builder.py USER_TEMPLATE 末尾放 vision_placeholder）

---

### 🟡 P3：final_projection 前缺少 LayerNorm

**问题：** LLM last-token hidden state 数值分布随输入变化大，直接做 Linear(3584→1024) 嵌入空间不稳定。

**修复：** 在 `_pool_hidden_state` 后、`final_projection` 前加 `nn.LayerNorm(llm_dim)`。

**文件：** `mts_agent/models/mts_embedder.py`
**状态：** ✅ 已实施（v12，`self.final_norm = nn.LayerNorm(llm_dim)`，dtype-aware cast）

---

### 🟡 P4：Pooling 策略 —— ts_tokens mean pooling

**问题：** `embedding_pooling="last"` 有 recency bias（只依赖最后一个 token）；研究表明 mean pooling > last-token 对 decoder embedding。对于多模态序列，对 TS token 区域做 mean pooling 可降低文本 token 的干扰。

**修复：** 改用 `embedding_pooling="ts_tokens"`（对 TS token 位置做 mean pooling），已在代码中实现，仅配置文件改动。

**文件：** `mts_agent/configs/natops_v12_arch.json`
**状态：** ✅ 已实施（v12，`embedding_pooling="ts_tokens"`；v11 的 `last` 因 vision_end 稀疏标记导致 LOO 跌至 56.67%，已修正）

---

### 🟡 P5：Projector 过重（128→7168→3584）

**问题：** 中间层 7168 = 2×llm_dim 相对于 128-dim 输入严重过宽（~26M 参数），在 180 样本上有过拟合风险；residual 和第一个 linear 路径冗余。

**修复：** 将 expansion_ratio 从 config 控制，默认改为 `hidden_dim = 512`（4×ts_hidden_dim），减少到 ~2M 参数：
```
128 → 512 → 3584
```

**文件：** `mts_agent/models/projector.py`, `mts_agent/config.py`
**状态：** ✅ 已实施（v13，`projector_hidden_dim=512`；projector.py 新增 `hidden_dim` 参数覆盖 expansion_ratio）

---

### 🟡 P6：Cross-Channel Mixer（跨通道信息融合）

**问题：** CNN TSEncoder 完全 channel-independent，没有跨通道交互。NATOPS 手势的双手协调模式（左手+右手空间关系）被完全忽略。

**修复：** 在 TSEncoder 输出之后加轻量 **Cross-Channel Attention**：
- 输入：`[B, C*P, D]`（C=24通道，P=6 patches，D=128）
- reshape 到 `[B*P, C, D]`，对 C 个 channel 做 MultiheadAttention（4 heads）
- 残差 + LayerNorm，reshape 回 `[B, C*P, D]`
- 参数量：~12K（可忽略不计）

**文件：** `mts_agent/models/ts_encoder.py`（新增 `ChannelMixer`），`mts_agent/models/mts_embedder.py`
**状态：** ✅ 已实施（v13，`channel_mixer=true, channel_mixer_heads=4`）

---

### 🟢 P7：SoftCLT 距离改进（多维 vs channel-mean 1D）

**问题：** 当前 SoftCLT 用 `ts.mean(axis=1)` 压成 1D 再算 Euclidean，丢失所有通道结构信息。

**修复：** 直接对原始 `[C, T]` 展平做 Euclidean（或用 DTW 近似），权重计算更准确。

**文件：** `mts_agent/engine/trainer.py`
**状态：** ✅ 已实施（v12，C×T flatten + L2 normalize，dist ∈ [0,2]）

---

**问题：** ProtoNCE（6 类 softmax）和 InfoNCE（108 个 gallery 负样本）共享同一可学习温度，最优值不同。

**修复：** 为 ProtoNCE 加独立 `proto_log_tau` 参数。

**文件：** `mts_agent/engine/trainer.py`
**状态：** ⬜ 待实施
